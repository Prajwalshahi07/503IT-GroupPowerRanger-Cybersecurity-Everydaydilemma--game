Our game title is:
Everyday Dilemma
The game is designed to present players with realistic online situations where they must make safe decisions related to cybersecurity and phishing awareness.
Our target audience is:
University students aged 18–25.
We selected this age group because university students frequently use emails, social media, online banking, and digital learning platforms, making them more vulnerable to phishing scams and cyber threats.
Our cyber-safety topic is:
Phishing awareness.
The main goal of the game is to help players identify fake emails, suspicious links, scam messages, and online fraud attempts while learning how to stay safe online.

Today we compared these tools:

HTML, CSS, and JavaScript
Canva
Scratch

We explored the strengths and limitations of each tool for creating our educational cybersecurity game.

We selected this tool because:
HTML, CSS, and JavaScript provide more flexibility and control for developing an interactive and professional-looking game.

HTML helps structure the game content.
CSS is used for styling and visual design.
JavaScript allows us to add interactivity, decision-making scenarios, scoring, and feedback systems.

These tools also make it easier to expand the project in the future.

Our first prototype will include:

At least one complete phishing scenario
A simple and clean game interface
Basic navigation and interaction features
Educational feedback after each choice
Cybersecurity explanations about phishing attacks and online safety

The prototype will focus on demonstrating the core gameplay and educational purpose of the project.
