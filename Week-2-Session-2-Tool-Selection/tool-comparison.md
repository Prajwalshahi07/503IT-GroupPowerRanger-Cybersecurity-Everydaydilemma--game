Tool 1: HTML / CSS / JavaScript
What we could build with it:
Web page structures and layouts
Web-based games
Phishing-style emails and login pages (simulation/educational use)
Interactive web pages using JavaScript + CSS
Strengths:
Runs directly in browsers (no downloads needed)
Easy to combine HTML, CSS, and JavaScript for interactivity
Good for creating realistic web pages and user interfaces
Suitable for simulating websites users are familiar with (e.g., university portals)
Limitations:
Simple HTML alone can look basic or static
CSS can become complex for advanced designs or graphics
Requires careful structure for larger projects
Suitable for target audience?
Yes
Familiar format for university students (websites, emails, login pages)
Can create realistic and believable web simulations

Tool 2: Canvas
What we could build with it:
2D animations and transitions
Interactive games with movement
Cybersecurity-themed visual environments
Dynamic visual feedback systems
Strengths:
Smooth animations and real-time movement
Good for interactive and immersive experiences
Useful for visual feedback (score systems, pop-ups, progress bars)
Limitations:
Limited to mostly 2D graphics (not advanced 3D)
Requires full JavaScript coding (no visual builder)
Time-consuming to create interactions and animations
Suitable for target audience?
No
Optional Tool 3: Scratch
What we could build with it:
Simple educational games
Basic interactive stories
Beginner-level cybersecurity simulations
Strengths:
Very easy to learn (drag-and-drop coding)
No need for coding syntax
Good for quick prototyping
Limitations:
Limited advanced features
Less professional appearance compared to web-based tools
Not suitable for complex or realistic simulations
Suitable for target audience?
No
