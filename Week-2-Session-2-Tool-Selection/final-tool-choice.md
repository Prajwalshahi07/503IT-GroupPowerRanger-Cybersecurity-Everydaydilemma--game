Section: Tool Selection & Justification

What to write: Final tool decision with reasoning, rejected alternatives, and prototype status for the cybersecurity awareness game project.
Final selected tool: HTML, CSS, and JavaScript
Reason for choice:
Game Fit: HTML and CSS (with JavaScript) excel at creating realistic email & SMS interfaces, hover effects, countdown timers, mission hub navigation, and a fully responsive mobile-first design using Coventry purple + neon cyberpunk theme.
Team Skills: As Computer Science and Cyber Security students, we already have strong foundational knowledge in HTML, CSS, and JavaScript, allowing us to build a high-quality, interactive prototype quickly and efficiently.
Target Audience: University students use browsers daily. A web-based game requires no installation, runs perfectly on both phones and laptops, and feels natural and accessible to our target users.
Available Time: We can rapidly develop and iterate a full clickable prototype (including Mission Hub + 3 complete scenarios) within the limited timeframe, which is ideal for upcoming student testing and feedback.

Rejected tool: Canva and Scratch
Reason for rejection:
Canva is excellent for quick static mock-ups but lacks the ability to create interactive elements such as functioning timers, scoring systems, clickable choices, or dynamic content.
Scratch is limited to block-based coding and cannot produce realistic university email/SMS interfaces or deliver a professional, polished web experience.
Neither tool supports a browser-based, mobile-friendly, and fully interactive game that meets our project requirements for realism and engagement.
